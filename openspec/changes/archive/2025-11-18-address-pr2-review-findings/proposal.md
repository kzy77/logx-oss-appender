# 变更：修复 PR #2 安全审查发现的问题

## Why
- PR #2 审查指出：分片上传在消费者线程阻塞、占位符解析静默回落为空字符串、丢弃路径日志/转储不安全，可能导致数据泄露或注入。
- 需要制定有边界的修复方案，清理上述风险，避免过度设计。

- 分片上传改为非阻塞、有限并发并设置超时（`storage.uploadTimeoutMs`，默认 30s / 30000ms），上传可复用消费者线程池但不得在当前消费线程内阻塞。
- 对无默认值的配置占位符解析失败时立即报错，禁止静默替换为空，并在 `applyXmlConfig` 之后执行必填配置自检，发现为空立即告警并终止启动。
- 丢弃路径的诊断日志做脱敏与限频；在转发日志到外部存储前执行载荷校验/清洗，降低泄露与注入风险。
- 为上述行为补充针对性的回归测试与文档。

## Impact
- 相关规范：oss-logging-safety
- 涉及代码：`logx-producer`（EnhancedDisruptorBatchingQueue、ConfigManager）、各日志框架 appender（Logback/Log4j/Log4j2）及共享异步引擎边界
